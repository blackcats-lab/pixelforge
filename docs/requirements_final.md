# 画像リサイズWebアプリ 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名

**PixelForge** — ブラウザ完結型 画像リサイズWebアプリケーション

### 1.2 目的

ブラウザ上で画像のアップスケーリング（拡大）およびダウンスケーリング（縮小）をクライアントサイドのみで完結させるWebアプリケーション。サーバーへの画像アップロードは一切行わず、Canvas APIによる補間処理ですべてのリサイズをブラウザ内で実行する。Vercelにデプロイし、静的サイト同等の高速配信を実現する。

### 1.3 アーキテクチャ方針

```
┌─────────────────────────────────────────────┐
│  ブラウザ（クライアント）                       │
│                                             │
│  File API → Image → Canvas API → Data URL   │
│  (読込)    (デコード) (リサイズ)    (出力)      │
│                                             │
│  ※ 画像データはブラウザ外に一切送信されない       │
└─────────────────────────────────────────────┘
         ↑ 配信のみ
┌─────────────────────────────────────────────┐
│  Vercel (Next.js)                            │
│  HTML + JS + CSS のみ配信                     │
│  サーバーサイド処理なし・API Route なし          │
└─────────────────────────────────────────────┘
```

### 1.4 リサイズ方式

| 方式 | 技術 | 説明 |
|------|------|------|
| 標準リサイズ | Canvas API `drawImage()` | `imageSmoothingQuality: 'high'` による高品質補間 |
| 多段階リサイズ | Canvas API 多段適用 | 4x以上の拡大時に2xずつ段階的にリサイズ |
| ピクセルアート対応 | Canvas API | `imageSmoothingEnabled: false` でNearest Neighbor相当 |

**AIモデル・サーバーサイド処理は使用しない。**

## 2. ユーザー要件

### 2.1 ターゲットユーザー

| カテゴリ | 想定ユーザー |
|---------|------------|
| 一般ユーザー | SNS投稿用に画像サイズを手軽に調整したい人 |
| クリエイター | イラスト・写真の解像度変更を行いたい人 |
| ビジネスユーザー | プレゼン資料・Web素材のサイズを統一したい人 |
| プライバシー重視 | 画像をサーバーにアップロードしたくない人 |

### 2.2 ユースケース

1. **UC-01**: 1枚の画像をアップロードし、指定サイズにリサイズしてダウンロードする
2. **UC-02**: アスペクト比を維持したまま幅または高さを指定してリサイズする
3. **UC-03**: プリセット倍率（0.25x / 0.5x / 2x / 4x）でワンクリックリサイズする
4. **UC-04**: SNSプリセット（Instagram, X/Twitter等）でサイズ指定する
5. **UC-05**: リサイズ前後の画像をスライダーで比較確認する
6. **UC-06**: 出力フォーマット（PNG / JPEG / WebP）と品質を指定する
7. **UC-07**: 複数画像を一括でリサイズする（クライアント内バッチ処理）
8. **UC-08**: ピクセルアート画像をドットが潰れないようにリサイズする

## 3. 機能要件

### 3.1 コア機能

#### F-01: 画像アップロード

- ドラッグ&ドロップおよびファイル選択ダイアログに対応
- 対応フォーマット: JPEG, PNG, WebP, BMP, GIF（静止画として）, SVG（ラスタライズ）
- 最大ファイルサイズ: 制限なし（ブラウザメモリ依存、目安50MB）
- アップロード時にサムネイルプレビューとメタデータ（元サイズ・ファイルサイズ・形式）を表示
- 複数ファイル同時選択対応（バッチ処理用、最大20枚）
- 入力画像のフォーマットに応じてデフォルト出力フォーマットを自動設定

#### F-02: リサイズパラメータ設定

- **ピクセル指定**（幅 × 高さ）
- **アスペクト比ロック/アンロック**
  - ロック時: 幅変更で高さ自動計算（`Math.round(width / aspectRatio)`）
  - ロック時: 高さ変更で幅自動計算（`Math.round(height * aspectRatio)`）
  - アンロック→ロック時: 現在の入力値からアスペクト比を再計算
- **倍率プリセットボタン**: 0.25x / 0.5x / 0.75x / 1.5x / 2x / 3x / 4x
- **SNSプリセット**:
  - Instagram 投稿: 1080 × 1080
  - Instagram ストーリー: 1080 × 1920
  - X/Twitter 投稿: 1200 × 675
  - X/Twitter ヘッダー: 1500 × 500
  - Facebook カバー: 820 × 312
  - YouTube サムネイル: 1280 × 720
  - OGP 画像: 1200 × 630
- **リサイズモード**:
  - Smooth（デフォルト）: `imageSmoothingEnabled: true`, `imageSmoothingQuality: 'high'`
  - Pixel Perfect: `imageSmoothingEnabled: false`（ピクセルアート向け）

#### F-03: Canvas APIリサイズ処理

```
処理フロー:
1. Image オブジェクトに Data URL をロード
2. Canvas を生成し、targetWidth × targetHeight に設定
3. ctx.imageSmoothingEnabled = true
4. ctx.imageSmoothingQuality = 'high'
5. ctx.drawImage(img, 0, 0, targetWidth, targetHeight)
6. canvas.toDataURL(format, quality) でリサイズ済み画像を取得
```

追加拡張:
- **多段階リサイズ**: 4x以上の拡大時は一括ではなく2xずつ段階的にCanvas描画を繰り返す
- **ピクセルアートモード**: `imageSmoothingEnabled = false` に切り替え
- **Web Worker対応（大画像用）**: OffscreenCanvasによるメインスレッド非ブロック処理

#### F-04: 出力フォーマット・品質設定

- フォーマット選択: PNG / JPEG / WebP
- JPEG / WebP 品質スライダー: 10% ～ 100%
- フォーマット変更時にリアルタイムで推定ファイルサイズ表示

#### F-05: プレビュー表示

- リサイズ前/後の画像切り替え表示
- チェッカーボード背景による透過表示
- ズーム・パン操作対応
- ビフォー/アフタースライダー比較ビュー

#### F-06: ダウンロード

- ファイル名: `{元ファイル名}_{幅}x{高さ}.{拡張子}`
- Data URL → `<a>` タグの `download` 属性でトリガー
- バッチ処理時はJSZipによるZIPアーカイブ生成・ダウンロード

#### F-07: バッチ処理（クライアント内）

- 複数画像の一括アップロード
- 共通パラメータ（倍率・フォーマット・品質）の一括適用
- 個別画像ごとのパラメータ上書き
- 処理進捗のリスト表示（各画像のステータス）
- 全画像完了後にZIPダウンロード

### 3.2 補助機能

#### F-08: 画像情報パネル

- ファイル名、ファイルサイズ（KB/MB表示）
- 元の幅 × 高さ
- リサイズ後の幅 × 高さ（リアルタイム更新）
- 拡大/縮小率（%表示）
- 推定出力ファイルサイズ

#### F-09: カスタムプリセット管理

- ユーザー定義プリセットの保存（localStorage）
- プリセットの編集・削除
- プリセット名 + 幅 × 高さ の登録

#### F-10: テーマ切り替え

- ライトモード / ダークモード
- OS設定に連動 + 手動切り替え
- `localStorage` でテーマ設定を永続化

#### F-11: リセット機能

- 全状態をクリアしてアップロード画面に戻る

## 4. 非機能要件

### 4.1 パフォーマンス

| 指標 | 目標値 |
|------|-------|
| 標準リサイズ（1024×1024 → 2048×2048） | 1秒以内 |
| 多段階リサイズ（512×512 → 4096×4096, 8x） | 3秒以内 |
| ダウンスケール（4096×4096 → 1024×1024） | 0.5秒以内 |
| バッチ処理（10枚, 2x） | 10秒以内 |
| 初回ロード（LCP, Vercel Edge） | 1.5秒以内 |
| JSバンドルサイズ（gzip） | 150KB以下 |

### 4.2 プライバシー・セキュリティ

- **画像データはサーバーに一切送信されない**（最大のセールスポイント）
- すべての処理がブラウザ内で完結
- ファイル拡張子とMIMEタイプの検証
- Canvas taint対策（CORS画像の扱い）
- XSSを防ぐData URL生成時のサニタイゼーション

### 4.3 ブラウザ互換性

| ブラウザ | バージョン | Canvas最大サイズ目安 |
|---------|-----------|-------------------|
| Chrome | 最新2バージョン | 16384 × 16384 |
| Firefox | 最新2バージョン | 11180 × 11180 |
| Safari | 最新2バージョン | 16384 × 16384 |
| Edge | 最新2バージョン | 16384 × 16384 |
| モバイル Safari | iOS 16+ | 4096 × 4096 |
| モバイル Chrome | Android 最新 | 環境依存 |

Canvas最大サイズを超えるリサイズ要求時は警告を表示する。

### 4.4 ユーザビリティ

- レスポンシブデザイン（モバイル/タブレット/デスクトップ）
- ダークモード対応
- 日本語 / 英語の多言語対応
- キーボードショートカット（Enter: リサイズ実行、Ctrl+S: ダウンロード）
- アクセシビリティ（WCAG 2.1 AA準拠）
- PWA対応（オフラインでも利用可能）

## 7. Canvas リサイズ処理仕様

### 7.1 標準リサイズ

```
入力: imageSrc (Data URL), targetWidth, targetHeight, format, quality
処理:
  1. new Image() に imageSrc をセット
  2. canvas = document.createElement('canvas')
  3. canvas.width = targetWidth, canvas.height = targetHeight
  4. ctx = canvas.getContext('2d')
  5. ctx.imageSmoothingEnabled = true
  6. ctx.imageSmoothingQuality = 'high'
  7. ctx.drawImage(img, 0, 0, targetWidth, targetHeight)
  8. return canvas.toDataURL(format, quality)
```

### 7.2 多段階リサイズ（4x以上の拡大）

一括リサイズだとCanvas APIの補間品質が低下するため、段階的に拡大する。

```
入力: imageSrc, targetWidth, targetHeight
判定: scale = max(targetWidth/origWidth, targetHeight/origHeight)
      scale > 4.0 の場合に多段階処理を適用

処理:
  1. currentCanvas = 元画像をCanvasに描画
  2. while (currentSize < targetSize):
       nextWidth = min(currentWidth * 2, targetWidth)
       nextHeight = min(currentHeight * 2, targetHeight)
       newCanvas = 新規Canvas(nextWidth, nextHeight)
       ctx.imageSmoothingEnabled = true
       ctx.imageSmoothingQuality = 'high'
       ctx.drawImage(currentCanvas, 0, 0, nextWidth, nextHeight)
       currentCanvas = newCanvas
  3. 最終ステップで目標サイズぴったりに調整
  4. return canvas.toDataURL(format, quality)
```

### 7.3 ピクセルアートモード

```
ctx.imageSmoothingEnabled = false
→ Nearest Neighbor相当の補間でドットの境界を保持
```

## 8. 制約事項・前提条件

- すべての処理がクライアントサイド（ブラウザ内）で完結する
- 画像データはネットワーク上に一切送信されない
- サーバーサイド処理（API Route, Server Actions）は使用しない
- Canvas APIの最大サイズはブラウザ環境に依存する（上限超過時は警告表示）
- 大画像処理時はブラウザのメモリ使用量に注意が必要
- `canvas.toDataURL()` のフォーマット対応はブラウザ依存（WebP非対応ブラウザあり）
- ユーザー認証は実装しない
- デプロイ先: Vercel（無料プランで運用可能）
